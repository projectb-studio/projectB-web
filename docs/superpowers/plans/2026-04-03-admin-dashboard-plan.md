# Admin Dashboard 설계 플랜

> 2026-04-03 — 4/2 미팅 결과 반영

## 목표

대표님(비개발자)이 직접 쇼핑몰 콘텐츠를 관리할 수 있는 관리자 대시보드 구현.
기존 엑셀/스프레드시트 관리 방식에서 웹 기반 관리로 전환.

---

## 핵심 기능 (미팅에서 요청된 항목)

### 1. 히어로 섹션 편집
- 메인 히어로 이미지 또는 영상 업로드/교체
- 히어로 텍스트(메인 카피, 서브 카피) 수정
- CTA 버튼 텍스트/링크 수정
- 이미지/영상 전환 (이미지 ↔ 영상 모드)
- **DB 테이블:** `pb_hero_settings` (type, media_url, heading, subheading, cta_text, cta_link)

### 2. 브랜드 페이지 편집
- 브랜드 스토리 텍스트 수정
- 브랜드 페이지 이미지들 교체 (히어로 이미지, 소재 소개 이미지 등)
- 소재 카테고리 이름/설명 수정
- **DB 테이블:** `pb_brand_content` (section, title, content, image_url, sort_order)

### 3. 상품 상세페이지 편집
- 상품 등록/수정/삭제
- 상품 이미지 업로드 (Cloudflare R2)
- 상품 옵션 관리 (색상, 사이즈)
- 상세 설명 (리치 텍스트 에디터 or 마크다운)
- 배송/케어 정보 수정
- 가격/할인가 설정
- 태그/카테고리 지정
- 재고 관리
- **기존 테이블:** `pb_products`, `pb_product_options`, `pb_product_images`

## 기술 스택

```
프레임워크:  Next.js 14 App Router (메인 사이트와 동일)
인증:       Supabase Auth (admin role 체크)
DB:         Supabase (기존 pb_ 테이블 + 관리 전용 테이블)
이미지:     Cloudflare R2 (기존 인프라)
에디터:     Tiptap (리치 텍스트) 또는 마크다운
차트:       Recharts (매출 통계)
```

## 레포지토리

CLAUDE.md에 따르면 `admin-dashboard`는 별도 레포 (`projectb-studio/admin-dashboard`).
단, 초기에는 메인 사이트(`projectb-web`) 내 `/admin` 라우트로 구현 후, 규모가 커지면 분리하는 것이 효율적.

---

## 관리자 인증

- Supabase Auth의 user_metadata에 `role: "admin"` 추가
- 미들웨어에서 `/admin/*` 라우트 접근 시 admin 역할 체크
- 대표님 계정 1개만 admin 권한 (사장님 혼자 사용)

---

## 구현 순서 (권장)

### Phase 1: 기본 골격 + 상품 관리
1. `/admin` 레이아웃 (사이드바, 헤더)
2. 관리자 인증 (admin role 체크)
3. 상품 목록/등록/수정/삭제
4. 이미지 업로드 (R2 연동)
5. 상품 옵션 관리

### Phase 2: 콘텐츠 편집
6. 히어로 섹션 편집
7. 브랜드 페이지 편집
8. 공지사항 관리

### Phase 3: 주문/고객
9. 주문 목록/상세/상태 변경
10. 회원 목록
11. 리뷰/문의 답변

### Phase 4: 마케팅/통계
12. 쿠폰/이벤트 관리
13. 매출 통계 대시보드
14. 재고 관리

## 참고

- 대표님은 비개발자이므로 UI가 직관적이고 쉬워야 함
- 엑셀에서 전환하는 것이므로 테이블 형태 UI가 익숙할 수 있음
- 모바일에서도 접근 가능해야 함 (매장에서 스마트폰으로 관리)
