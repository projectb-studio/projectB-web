# 🛒 쇼핑몰 홈페이지/앱 제작 파이프라인 — Cowork 프롬프트

---

## 프로젝트 개요

나는 TypeScript/C++ 개발자이고, 쇼핑몰을 운영하는 사장님에게 홈페이지와 앱을 만들어주려고 해.
사장님은 웹/앱 개발을 전혀 모르는 비개발자야.

아래 4단계를 순서대로 진행해줘.
각 단계의 결과물은 **실제 파일**로 만들어줘 (React Artifact, DOCX, PDF 등).

---

## 1단계: 고객 인터뷰 체크리스트 제작

**목표:** 비개발자 사장님이 쉽게 이해하고 답할 수 있는 인터뷰 질문지를 만들어줘.

**요구사항:**
- 일반인이 이해할 수 있는 쉬운 용어로 질문 작성 (전문 용어 금지)
- 체크박스형(선택지) + 서술형(자유 입력) 혼합
- 질문 답변 결과로 웹/앱 전체 구조를 설계할 수 있어야 함
- 아래 8개 섹션을 포함해서 총 30~40개 질문으로 구성

**필수 섹션:**
1. **가게 소개** — 브랜드명, 상품 종류/수량, 타겟 고객, 오프라인 매장 여부, 차별점, 로고 유무
2. **디자인 & 분위기** — 원하는 무드(심플/고급/귀여움 등), 선호 색상, 참고 사이트, 상품 사진 스타일
3. **필요 페이지** — 메인/상품목록/상세/장바구니/결제/마이페이지/공지/고객센터/브랜드소개/블로그/리뷰 등 선택, 상품 카테고리 분류 방식
4. **필요 기능** — 결제(카드/카카오페이/네이버페이/토스 등), 회원(소셜로그인/등급/적립금/쿠폰), 상품(옵션/리뷰/Q&A/재입고알림/추천), 배송(택배/당일/추적/무료배송조건), 마케팅(배너/타임세일/알림톡/푸시/뉴스레터), 고객응대(1:1문의/채팅/카카오톡/FAQ/교환반품)
5. **관리자 기능** — 사장님이 직접 할 작업(상품관리/주문처리/매출통계/회원관리/쿠폰생성/재고관리), 관리자 수, 현재 관리 방식
6. **앱 관련** — 앱 필요 여부(안드로이드/iOS/PWA), 앱 전용 기능(푸시/QR/위치기반/카메라)
7. **운영 & 예산** — 도메인, 일정, 예산, 월 유지비, 현재 사용 플랫폼(스마트스토어/카페24 등), 데이터 이전 필요 여부
8. **추가 요청** — 자유 아이디어, 피하고 싶은 것, 경쟁사 정보

**산출물:**
- ✅ 인터랙티브 체크리스트 (React .jsx) — 사장님이 직접 체크하고 결과를 JSON으로 내보내기 가능
- ✅ 프린트용 문서 (Word .docx + PDF) — 표지 + 목차 + 섹션별 구분, 체크박스(☐)와 작성란 포함

## 2단계: 체크리스트 응답 기반 설계 (사장님 답변 입력 후 진행)

**목표:** 1단계에서 받은 답변을 기반으로 Relume 스타일의 설계 산출물 4종을 생성해줘.

### 2-1. Sitemap (사이트맵)

**요구사항:**
- 체크리스트 응답에서 선택한 페이지들을 기반으로 사이트 구조도 생성
- 페이지 간 계층 관계 (메인 → 카테고리 → 상세 등) 시각화
- 네비게이션 구조 포함
- SVG 다이어그램 또는 인터랙티브 React Artifact로 제작
- 사장님이 한눈에 "우리 홈페이지에 이런 페이지들이 있구나" 이해할 수 있게

### 2-2. Wireframe (와이어프레임)

**요구사항:**
- 주요 페이지별 레이아웃을 로우파이(lo-fi) 와이어프레임으로 제작
- 최소 아래 페이지는 포함: 메인(홈), 상품 목록, 상품 상세, 장바구니, 마이페이지
- 각 페이지의 섹션 배치 (헤더, 히어로, 상품 그리드, CTA, 푸터 등) 표현
- React Artifact로 클릭하면 페이지 간 이동 가능한 인터랙티브 프로토타입
- 모바일/데스크톱 전환 토글 포함

### 2-3. Style Guide (스타일 가이드)

**요구사항:**
- 체크리스트의 디자인 섹션 답변 기반으로 생성
- 포함 항목:
  - Color Palette: Primary, Secondary, Accent, Neutral, 상태색(Success/Warning/Error)
  - Typography: 헤딩(H1~H4), 본문, 캡션 — 한글 폰트 포함
  - Buttons: Primary, Secondary, Outline, Ghost, Disabled 상태
  - Form Elements: Input, Select, Checkbox, Radio
  - Spacing & Border Radius 규칙
  - 아이콘 스타일
- React Artifact로 실제 컴포넌트 미리보기 가능하게

### 2-4. Design (데모 프로토타입)

**요구사항:**
- Style Guide를 적용한 하이파이(hi-fi) 데모 프로토타입
- 최소 메인 페이지 + 상품 상세 페이지는 실제 디자인 수준으로
- 사장님이 "이런 느낌으로 만들어지는구나!" 바로 이해할 수 있는 수준
- React Artifact로 제작, 실제 클릭/스크롤 가능
- 더미 상품 데이터 포함 (사장님 업종에 맞는)
- 반응형 (모바일/데스크톱)

## 3단계: 기술 스택 & 구현 설계

**목표:** 2단계 설계를 실제로 구현하기 위한 기술 문서 작성

**포함 내용:**
- 추천 기술 스택 (프론트엔드/백엔드/DB/인프라)
- 프로젝트 폴더 구조
- 주요 API 엔드포인트 설계
- DB 스키마 (ERD)
- 결제 연동 방안 (PG사 선택)
- 배포 전략 (Vercel/AWS 등)
- 예상 개발 일정 (간트 차트 또는 로드맵)

## 4단계: Notion 프로젝트 보드 세팅 (선택)

**목표:** 프로젝트 관리용 Notion DB 생성

**포함 항목:**
- 프로젝트 진행 상황 트래커 (Board 뷰: 기획 → 디자인 → 개발 → 테스트 → 배포)
- 사장님 요청사항 로그 DB
- 페이지별 개발 체크리스트
- 타임라인 뷰

## 작업 규칙

1. **각 단계 산출물은 실제 파일로 생성**해줘 (채팅에 텍스트로만 쓰지 말고)
2. **한국어**로 작성하되, 코드/기술 용어는 영어 유지
3. **사장님 관점의 산출물** (Sitemap, Wireframe, Style Guide, Demo)은 전문 용어 없이 직관적으로
4. **개발자 관점의 산출물** (기술 설계서)은 상세하고 구체적으로
5. 1단계 체크리스트를 먼저 만들고, 내가 사장님 답변을 입력하면 2~4단계를 순차 진행
6. 디자인은 generic AI 느낌 피하고, 실제 운영 가능한 수준의 퀄리티로
