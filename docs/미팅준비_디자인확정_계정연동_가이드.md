# PART 2: 서버/계정 연동 가이드

## 1. Supabase (DB + Auth + API)

### 1-1. 프로젝트 생성
1. https://supabase.com 접속 → 로그인 (GitHub 또는 이메일)
2. "New Project" 클릭
3. 설정:
   - **Organization**: projectb-studio (없으면 생성)
   - **Project name**: projectb-web
   - **Database Password**: 강력한 비밀번호 생성 후 **반드시 저장**
   - **Region**: Northeast Asia (Tokyo) — 한국에서 가장 가까움
   - **Plan**: Free tier (시작용, 나중에 Pro 업그레이드)
4. 생성 후 **Project Settings → API** 에서:
   - `Project URL` → `.env.local`의 `NEXT_PUBLIC_SUPABASE_URL`에 입력
   - `anon public key` → `.env.local`의 `NEXT_PUBLIC_SUPABASE_ANON_KEY`에 입력

### 1-2. 소셜 로그인 설정 (Authentication → Providers)

#### Kakao 로그인
1. https://developers.kakao.com → 대표님 카카오 계정으로 로그인
2. "내 애플리케이션" → "애플리케이션 추가"
   - 앱 이름: PROJECT B
   - 사업자명: (대표님 사업자명)
3. 생성 후 **앱 키** 탭에서 "REST API 키" 복사
4. **카카오 로그인** 메뉴 → 활성화 ON
5. **Redirect URI** 추가: `https://<supabase-project-id>.supabase.co/auth/v1/callback`
6. **동의항목** → 닉네임, 프로필 사진, 이메일 → 필수 동의로 설정
7. Supabase Dashboard → Authentication → Providers → Kakao:
   - Client ID: REST API 키
   - Client Secret: (앱 키 탭의 Secret Key, 없으면 보안 탭에서 생성)
   - 활성화 ON

#### Naver 로그인
1. https://developers.naver.com → 대표님 네이버 계정으로 로그인
2. Application → 애플리케이션 등록
   - 애플리케이션 이름: PROJECT B
   - 사용 API: 네이버 로그인 (회원이름, 이메일, 프로필 사진)
   - 로그인 오픈 API 서비스 환경: WEB
   - 서비스 URL: `https://projectb.kr` (확정 후)
   - Callback URL: `https://<supabase-project-id>.supabase.co/auth/v1/callback`
3. 생성 후 Client ID / Client Secret 복사
4. Supabase Dashboard → Authentication → Providers → 커스텀 OIDC 또는 Naver 설정

#### Google 로그인
1. https://console.cloud.google.com → 대표님 구글 계정
2. 새 프로젝트 생성: "PROJECT B"
3. APIs & Services → Credentials → Create Credentials → OAuth client ID
   - Application type: Web application
   - Authorized redirect URIs: `https://<supabase-project-id>.supabase.co/auth/v1/callback`
4. Client ID / Client Secret 복사
5. APIs & Services → OAuth consent screen 설정 (앱 이름, 로고 등)
6. Supabase Dashboard → Authentication → Providers → Google:
   - Client ID / Secret 입력, 활성화

#### Apple 로그인
1. https://developer.apple.com → Apple Developer Program 가입 필요 (연 $99)
2. Certificates, Identifiers & Profiles → Identifiers → App IDs
3. Sign In with Apple 활성화
4. Services ID 생성 → Redirect URL 설정
5. Key 생성 → .p8 파일 다운로드
6. Supabase에서 Apple provider 설정
> ⚠️ Apple Developer 가입비가 있으므로 **초기 출시에서 제외하고 나중에 추가 권장**

### 1-3. DB 스키마 생성
- Supabase Dashboard → SQL Editor에서 마이그레이션 실행
- 테이블 목록: `pb_products`, `pb_orders`, `pb_cart`, `pb_reviews` 등
- RLS(Row Level Security) 정책 설정 필수
- **이건 미팅 후 개발자가 처리** (대표님 작업 아님)

## 2. Cloudflare R2 (이미지 저장소)

### 2-1. 계정 생성 및 R2 활성화
1. https://dash.cloudflare.com → 계정 생성 (대표님 이메일)
2. 좌측 메뉴 → **R2 Object Storage** → "Purchase R2 Plan" (무료 10GB/월)
3. 결제 수단 등록 (초과분만 과금, 기본 무료)

### 2-2. 버킷 생성
1. R2 → Create bucket
   - **Bucket name**: `projectb-images`
   - **Location**: APAC (아시아태평양)
2. 생성 후 Settings → Public access → **Enable** (이미지 공개 접근용)
3. Custom domain 설정 (선택): `images.projectb.kr` 같은 서브도메인

### 2-3. API 토큰 생성
1. R2 → Manage R2 API Tokens → Create API token
   - Token name: `projectb-upload`
   - Permissions: Object Read & Write
   - Specify bucket: `projectb-images`
2. 생성 후 저장:
   - Account ID → `.env.local`의 `R2_ACCOUNT_ID`
   - Access Key ID → `R2_ACCESS_KEY_ID`
   - Secret Access Key → `R2_SECRET_ACCESS_KEY`
3. Public URL → `NEXT_PUBLIC_R2_PUBLIC_URL`
